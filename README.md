# Mirror Master

Mirror Master 是一个纯浏览器、本地运行的 Pixelanim 网格镜像工具。不同图片可以有不同的自然像素尺寸、网格原点、逻辑列数、逻辑行数和整数正方形单元尺寸。图片不会上传到服务器，也不依赖后端、OCR、分析服务或远程运行时。

## 使用流程

1. 上传一张 Pixelanim PNG、JPEG 或 WebP 图片。
2. 应用在图片自然分辨率下自动识别主网格的真实横向、纵向边界。
3. 检查原图上的边界叠加层。状态栏会显示识别到的列数、行数和单元像素尺寸。
4. 如果识别范围不完整，可移动搜索区域，或拖动其边缘与角点。拖动时显示原始搜索区域；松开后应用仅在该区域内重新检测，并将叠加层吸附到真实网格边界。
5. 点击“生成镜像”查看结果；需要修改时点击“返回调整”。结果生成后可点击“下载结果 PNG”保存本地 PNG 文件。

## 网格识别

应用内置本地 OpenCV.js 运行时，不从 CDN 加载代码。检测器在搜索区域内读取原始自然像素，转为灰度图并进行自适应阈值处理，再通过形态学分别提取纵向和横向线段。只有线段发生明显中断、主检测无法形成有效网格时，才使用 `HoughLinesP` 补充线段证据。

线掩码会投影到 X、Y 轴，并把连续线像素聚合为整数边界中心。检测器同时比较基础间距及其二至六倍谐波，选择能够解释两轴边界的最小稳定正方形格距，再将所有边界吸附到同一个正整数单元尺寸。列数和行数分别由 `xBoundaries.length - 1` 与 `yBoundaries.length - 1` 得出，不使用固定行列数或默认短轴格数。

## 镜像方式

镜像结果与原图尺寸完全相同。应用先复制完整原图，再按照已确认的 `xBoundaries` 和 `yBoundaries` 逐格搬运像素。每个源列会移动到对称目标列，源矩形与目标矩形的宽高都必须严格等于同一个 `cellSize`。

单元内部不会翻转、缩放、重建、OCR 或重绘，因此 H7、A17、B13 等内部标签仍保持正向可读。检测网格以外的像素保持原样。PNG 下载通过浏览器 Canvas 的 `toBlob()` 在本地生成。

重新识别、调整搜索区域、更换图片或关闭页面时，应用会取消过期任务并清理旧的结果下载地址。同一个文件可以重复选择。

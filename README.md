# Mirror Master

Mirror Master 是一个面向 Pixelanim 风格网格图片的前端项目。当前版本完成了 GRID-001：选择一张本地图片，显示原图预览与真实元数据，并在浏览器本地尝试自动检测完整的 34 × 27 主网格。

## 项目目的

Mirror Master 计划成为一个完全在浏览器本地运行的工具，用于处理由规则网格组成的 Pixelanim 风格图片。当前自动检测原型只识别固定 34 列 × 27 行的主网格结构。未来目标是基于检测到的网格单元，把单元格的位置关系做镜像变换，同时保留单元格内部的可读标签。

典型场景是：图片里某个格子包含 `H7` 这样的标签。Mirror Master 计划移动这个格子在网格中的位置，但不把 `H7` 本身当作像素内容左右翻转成不可读的反向文字。

## 已完成能力：本地图片输入、原图预览与自动检测原型

当前应用支持：

- 通过可见的点击区域选择一张本地图片。
- 将一张本地图片拖放到指定区域。
- 只接受单个 PNG、JPEG 或 WebP 文件。
- 拒绝多文件拖放，并显示清晰的中文错误。
- 使用浏览器本地图片解码读取真实像素宽度和高度。
- 使用对象 URL 显示响应式原图预览，保持宽高比，不做不必要的放大。
- 显示真实文件名、格式、尺寸和人类可读文件大小。
- 支持更换图片、移除图片，并允许移除后再次选择同一个文件。
- 在图片解码完成后，自动尝试检测完整的 34 × 27 Pixelanim 主网格。
- 支持图片像素尺寸、外围留白、标题高度、坐标条和图例位置变化。
- 对高或中置信度结果显示只读叠加层，包括检测外框和全部 34 列 × 27 行分隔线。
- 对低置信度或无效候选显示清晰的中文失败状态，不显示误导性的叠加层。

## 使用步骤

1. 运行开发服务后打开页面。
2. 点击“选择一张本地图片”，或把单个图片文件拖放到该区域。
3. 查看原图预览、图片信息和自动检测状态。
4. 需要换图时点击“更换图片”；需要清空状态时点击“移除图片”。
5. 如果检测成功，请检查叠加层外框和分隔线是否贴合原图网格。

## 支持格式

- PNG：`image/png`
- JPEG：`image/jpeg`
- WebP：`image/webp`

其他格式会被拒绝。一次只能处理一张图片。

## 隐私与本地处理原则

- 当前图片输入、格式校验、图片解码和预览都在浏览器本地完成。
- 应用使用对象 URL 预览图片，不把图片转成 base64。
- 被替换、移除或页面卸载时，对象 URL 会被撤销。
- 页面没有后端、远程存储、分析埋点、远程图片、API key 或网络上传逻辑。
- 选中文件的字节和元数据不会由应用发送到网络。
- 自动检测使用浏览器本地 Canvas 2D 读取像素，只分析用户选择的本地文件。
- 检测不会使用 OCR、OpenCV、AI 服务、网络上传、远程图片或外部字体。

## 自动检测方法摘要

GRID-001 的检测逻辑位于 `src/features/grid-detection`，核心假设是主网格固定为 34 列 × 27 行，因此需要寻找 35 条垂直边界和 28 条水平边界。

检测流程：

1. 从原始本地 `File` 解码图片，使用自然像素尺寸，不读取 CSS 显示尺寸。
2. 将图片绘制到有上限的分析 Canvas，保留自然坐标与分析坐标的比例映射。
3. 计算灰度亮度和水平/垂直边缘证据。
4. 在一维投影中搜索周期性的 35 条垂直边界和 28 条水平边界。
5. 组合候选矩形，并约束单元格接近正方形、外框比例接近 34:27、候选区域不越界。
6. 在候选区域内聚合边界证据，并与单元中心基线证据对比。
7. 使用原始分辨率像素在候选外框附近做小窗口精修。
8. 将结果映射回自然图像坐标，并生成单调边界数组；首尾边界与检测外框完全一致。

检测不依赖文件名、标签文字、颜色语义或 OCR。由于彩色单元、黑色单元、抗锯齿、JPEG/WebP 压缩和局部遮挡会让单条线的可见性不同，评分会聚合完整候选区域的周期性证据，而不是要求每条线同等清晰。

## 置信度行为

检测评分由四部分组成：

- 几何：单元格近似正方形、外框比例接近 34:27、候选区域位于图片内。
- 周期：边界在固定 34 × 27 结构下的重复稳定性。
- 边界：预期网格线位置的聚合边缘强度。
- 对比：预期边界证据相对于单元中心基线证据的差异。

置信度分为中文“高”“中”“低”。高和中置信度会显示只读 SVG 叠加层；低置信度和无效候选会显示失败说明，不显示叠加层。检测结果仍是原型判断，直到后续实现用户校正前，都应人工检查叠加层是否对齐。

## 当前限制

当前版本仍未实现：

- 用户手动校正。
- OCR。
- OpenCV。
- 网格单元镜像。
- 图片导出。

当前自动检测仍有局限：

- 只针对完整的 34 × 27 主网格。
- 不支持手动框选、拖拽修正或自由裁剪。
- 如果图片没有稳定的周期性网格、主网格严重缺失、候选矩形比例异常，或边界证据弱于单元内部纹理，会拒绝显示叠加层。
- 检测结果尚未进入镜像处理流程。

也就是说，当前版本提供“本地选择、原图预览和自动检测原型”，不声称已经完成用户校正、镜像处理或导出。

## 整图像素翻转与网格单元位置镜像的区别

普通的整图像素翻转会把图片每一个像素按左右方向反转。这样虽然视觉上完成了镜像，但图片中的文字、标签和内部标记也会一起被反向翻转，例如 `H7` 会变成镜像文字。

Mirror Master 计划实现的是网格单元位置镜像：先理解图片由哪些网格单元组成，再把这些单元在网格坐标中的位置做镜像排列。每个单元内部的图像内容和标签保持原始方向，因此标签仍应可读。

# Mirror Master

Mirror Master 是一个面向 Pixelanim 风格网格图片的前端项目。当前版本完成了 GRID-PRECISION-001：选择一张本地图片，显示原图预览与真实元数据，在浏览器本地尝试自动检测完整的 34 × 27 主网格，在检测结果不可直接接受时保留用户网格粗校正 fallback，并把已应用的近似 `GridSelection` 精修为严格整数像素 `PixelGridCalibration` 候选。

## 项目目的

Mirror Master 计划成为一个完全在浏览器本地运行的工具，用于处理由规则网格组成的 Pixelanim 风格图片。当前实现只识别、粗校正和精修固定 34 列 × 27 行的完整主网格外框。未来目标是基于已确认的整数像素校准，把单元格的位置关系做镜像变换，同时保留单元格内部的可读标签。

典型场景是：图片里某个格子包含 `H7` 这样的标签。Mirror Master 计划移动这个格子在网格中的位置，但不把 `H7` 本身当作像素内容左右翻转成不可读的反向文字。

## 已完成能力：本地图片输入、自动检测、粗校正与整数精修

当前应用支持：

- 通过可见的点击区域选择一张本地图片。
- 将一张本地图片拖放到指定区域。
- 只接受单个 PNG、JPEG 或 WebP 文件。
- 拒绝多文件拖放，并显示清晰的中文错误。
- 使用浏览器本地图片解码读取真实像素宽度和高度。
- 使用对象 URL 显示响应式原图预览，保持宽高比，不做不必要的放大。
- 显示真实文件名、格式、尺寸和人类可读文件大小。
- 支持更换图片、移除图片，并允许移除后再次选择同一个文件。
- 在图片解码完成后，自动尝试检测完整的 34 × 27 Pixelanim 主网格。
- 支持图片像素尺寸、外围留白、标题高度、坐标条和图例位置变化。
- 对高或中置信度结果显示叠加层，并提供“使用检测结果”或“调整检测区域”两个中文动作。
- 对低置信度或无效候选显示清晰的中文失败状态，只提供“手动选择网格区域”，不会接受低置信度结果。
- 在检测成功时，可从检测到的自然坐标外框开始校正。
- 在检测失败时，必须由用户先拖出一个近似完整外框；应用不会凭空创建可接受区域。
- 支持四个角手柄、四个边手柄和整个矩形拖动，只编辑完整的轴对齐 34 × 27 外框。
- 支持 Fit、放大、缩小、100% 缩放，图片放大后预览区域可滚动。
- 显示自然坐标、单元尺寸、近似正方形校验状态和中文不匹配警告。
- 支持围绕中心将当前外框校正为精确 34:27，并保持在图片边界内。
- 支持应用、取消、重新开始、有检测结果时重置为检测结果，以及应用后的再次编辑。
- 应用后只在浏览器内存中创建 typed selection，来源为 `automatic` 或 `manual`。
- 更换或移除图片会立即清除已应用选择、校正状态、精修状态、计时器和过期结果。
- 已应用的近似 `GridSelection` 只代表粗校正，不代表镜像、导出、下载或处理就绪。
- 支持从原始本地 `File` 的自然分辨率像素中进行整数像素精修，不读取 CSS 显示尺寸。
- 精修候选固定为 34 列 × 27 行、单个正整数正方形单元尺寸、35 条整数垂直边界和 28 条整数水平边界。
- 精修候选必须由用户显式确认后，才会生成 `processingReady: true` 的 `PixelGridCalibration`。

## 使用步骤

1. 运行开发服务后打开页面。
2. 点击“选择一张本地图片”，或把单个图片文件拖放到该区域。
3. 查看原图预览、图片信息和自动检测状态。
4. 如果检测置信度为高或中，可点击“使用检测结果”，或点击“调整检测区域”后拖动手柄微调。
5. 如果检测失败或置信度低，点击“手动选择网格区域”，然后在图片上拖出完整 34 × 27 外框的近似区域。
6. 校正时检查自然坐标、单元尺寸和校验状态；如比例不匹配，可点击“校正为 34:27”。
7. 点击“应用”创建浏览器内存中的粗校正网格选择；点击“取消”返回上一个检测或已应用状态。
8. 在“整数像素精修”面板点击“自动精修”，检查 X、Y、单元、右边界、下边界、证据和就绪状态。
9. 如需微调，使用 X/Y/单元的 `-1`、`+1`、整数输入或键盘方向键；方向键每次 1 px，Shift + 方向键每次 10 px。
10. 只有候选有效时才能点击“确认精确网格”；确认后才会显示 `processingReady: true`。
11. 需要换图时点击“更换图片”；需要清空状态时点击“移除图片”。

## 粗校正与精修流程

粗校正 fallback 只处理完整的轴对齐 34 × 27 外框。它不会旋转图片、做透视校正、编辑自由四边形、任意裁剪、OCR、OpenCV、后端上传、分析埋点、镜像、导出或下载。

粗校正使用 12% 单元近似正方形容差，只用于让用户得到可编辑的近似外框。这个容差永远不代表镜像、导出、下载或处理就绪。

检测成功时：

- 高或中置信度结果不会被自动应用。
- 用户必须明确点击“使用检测结果”，才会创建来源为 `automatic` 的选择。
- 用户也可以点击“调整检测区域”，以检测到的自然坐标矩形作为起点进行手动校正。

检测失败或低置信度时：

- 低置信度永远不会被接受为自动选择。
- 页面只提供“手动选择网格区域”。
- 用户必须在图片上拖出一个近似完整外框，手柄才会出现。
- 如果初始拖拽区域太小，会被拒绝并要求重新拖拽。

校正编辑器支持：

- 四个角手柄：同时改变两个相邻边。
- 四个边手柄：只改变对应的一条边。
- 整个矩形拖动：平移完整外框。
- 应用：当前区域通过粗校验后创建来源为 `manual` 的近似选择。
- 取消：放弃当前草稿，返回已应用选择或自动检测预览。
- 重新开始：清除当前草稿，重新拖出手动外框。
- 重置为检测结果：当存在高或中置信度检测结果时，回到该检测矩形。
- 编辑已应用区域：应用后仍可再次进入校正。

## 整数像素精确校准

精修阶段把已应用的近似 `GridSelection` 转换为严格的 `PixelGridCalibration` 候选。精修只在浏览器本地运行，输入是当前原始本地 `File`，分析尺寸是图片自然像素尺寸，不使用预览图 CSS 尺寸。

`PixelGridCalibration` 的合同是：

- `source` 继承粗选择来源：`automatic` 或 `manual`。
- `naturalImage` 保存自然图片宽高。
- `left`、`top`、`right`、`bottom` 必须都是整数自然像素。
- `cellSize` 必须是一个正整数，且横向和纵向共用同一个值。
- `columns` 固定为 `34`，`rows` 固定为 `27`。
- `right = left + 34 * cellSize`。
- `bottom = top + 27 * cellSize`。
- `verticalBoundaries` 固定 35 个整数，`horizontalBoundaries` 固定 28 个整数。
- 所有边界必须严格单调、等距、在图片内，且不能包含小数。
- `evidence` 保存边界强度、边界相对单元中心对比、周期一致性、外缘支持、总分和候选差距。
- 只有用户点击“确认精确网格”后才会生成 `processingReady: true`；候选状态不会自动确认。

精修状态使用中文显示：

- `idle`：空闲。
- `refining`：精修中。
- `candidate`：候选。
- `confirmed-ready`：已确认可处理。
- `rejected`：已拒绝。

## 精修算法摘要

精修算法使用 Canvas 2D 在原始分辨率读取亮度和边缘证据：

1. 从当前原始本地 `File` 解码图片，并确认自然尺寸与当前粗选择一致。
2. 从粗选择的宽度 / 34 和高度 / 27 估计附近整数 `cellSize`。
3. 对每个整数 `cellSize`，只构造精确 `34 * cellSize` × `27 * cellSize` 的候选矩形。
4. 在粗矩形附近对整数 `left` 和 `top` 做有界粗到细搜索。
5. 每个候选都生成完整 35 条垂直边界和 28 条水平边界。
6. 使用全分辨率亮度边缘证据评分：预期边界证据、边界相对单元中心对比、周期一致性、外框边缘支持和候选差距。
7. 弱证据、候选差距过小、越界、边界不等距、非整数、周期不稳定或疑似非整数缩放都会被拒绝。

精修不会做镜像、导出、下载、OCR、OpenCV、后端上传、旋转、透视校正或自由裁剪。

## 精修控件

精修面板显示 X、Y、单元尺寸、右边界、下边界、网格大小、证据和就绪状态，并提供：

- 自动精修：从当前粗选择搜索整数像素候选。
- X/Y/单元 `-1` 和 `+1`：每次调整一个自然像素或一个单元像素。
- 整数输入：直接输入 X、Y 或单元尺寸。
- 键盘方向键：当前输入每次变化 1 px；Shift + 方向键每次变化 10 px。
- 确认精确网格：只在候选通过严格合同与证据校验时可用。
- 取消精修：清除候选，保留粗选择。
- 返回粗校正：回到已应用粗选择的编辑状态，并清除精修就绪结果。

手动调整会重新生成全部边界并重新验证证据；无效候选不能确认。

## 坐标、缩放与校验

- 所有选择几何都保存在自然图片像素中，不使用 CSS 显示尺寸作为数据源。
- 图片和 SVG 叠加层放在同一个自然坐标 stage 中，SVG `viewBox` 与图片自然尺寸一致。
- Fit、放大、缩小和 100% 只改变 stage 的 CSS 尺寸；指针坐标会按 SVG 边界精确换算回自然像素。
- 粗校正渲染会从外框确定性生成 35 条垂直边界和 28 条水平边界，这些边界可以是近似小数。
- 精修渲染只使用整数边界，并与原图和 SVG 叠加层共享同一个自然坐标 `viewBox`。
- 外框会被夹在图片边界内，并强制最小尺寸为 136 × 108 自然像素，即每个单元至少 4 × 4 自然像素。
- 粗校正的近似正方形校验要求派生单元宽高偏差不超过 12%。超出容差时会显示中文不匹配警告，且不能应用。
- “校正为 34:27”会围绕当前中心点调整外框到精确 34:27，并在图片边界允许范围内完成。
- 只有精修确认后的 `PixelGridCalibration` 才会显示 `processingReady: true`。

## 可访问性与输入方式

- 校正使用 Pointer Events 和 pointer capture，支持鼠标、触控笔和触摸。
- 八个手柄和移动区域都可键盘聚焦，并带中文 `aria-label`。
- 方向键每次移动 1 个自然像素；Shift + 方向键每次移动 10 个自然像素。
- 边手柄只改变一条边，角手柄改变两条边，移动区域平移完整矩形。
- 手柄有可见焦点样式。
- 校正状态、错误和应用结果通过 polite live region 通知。
- 精修候选、拒绝、取消和确认状态也通过 polite live region 通知。
- 按钮和可点击目标满足至少 44 CSS 像素的命中尺寸；手柄的大命中区域是透明的，不遮挡网格线。

## 隐私与本地处理原则

- 当前图片输入、格式校验、图片解码、预览、自动检测、粗校正和整数精修都在浏览器本地完成。
- 应用使用对象 URL 预览图片，不把图片转成 base64。
- 被替换、移除或页面卸载时，对象 URL 会被撤销。
- 页面没有后端、远程存储、分析埋点、远程图片、API key 或网络上传逻辑。
- 选中文件的字节和元数据不会由应用发送到网络。
- 自动检测和整数精修使用浏览器本地 Canvas 2D 读取像素，只分析用户选择的本地文件。
- 粗校正和精修只记录浏览器内存中的自然坐标选择或校准，不写入本地文件或远程服务。
- 检测、校正和精修不会使用 OCR、OpenCV、AI 服务、网络上传、远程图片或外部字体。

## 自动检测方法摘要

GRID-CORRECTION-001 仍沿用 `src/features/grid-detection` 中的自动检测逻辑，核心假设是主网格固定为 34 列 × 27 行，因此需要寻找 35 条垂直边界和 28 条水平边界。GRID-PRECISION-001 另在 `src/features/grid-precision` 中实现严格整数像素精修。

检测流程：

1. 从原始本地 `File` 解码图片，使用自然像素尺寸，不读取 CSS 显示尺寸。
2. 将图片绘制到有上限的分析 Canvas，保留自然坐标与分析坐标的比例映射。
3. 计算灰度亮度和水平/垂直边缘证据。
4. 在一维投影中搜索周期性的 35 条垂直边界和 28 条水平边界。
5. 组合候选矩形，并约束单元格接近正方形、外框比例接近 34:27、候选区域不越界。
6. 在候选区域内聚合边界证据，并与单元中心基线证据对比。
7. 使用原始分辨率像素在候选外框附近做小窗口精修。
8. 将结果映射回自然图像坐标，并生成单调边界数组；首尾边界与检测外框完全一致。

检测不依赖文件名、标签文字、颜色语义或 OCR。由于彩色单元、黑色单元、抗锯齿、JPEG/WebP 压缩和局部遮挡会让单条线的可见性不同，评分会聚合完整候选区域的周期性证据，而不是要求每条线同等清晰。

## 置信度行为

检测评分由四部分组成：

- 几何：单元格近似正方形、外框比例接近 34:27、候选区域位于图片内。
- 周期：边界在固定 34 × 27 结构下的重复稳定性。
- 边界：预期网格线位置的聚合边缘强度。
- 对比：预期边界证据相对于单元中心基线证据的差异。

置信度分为中文“高”“中”“低”。高和中置信度会显示 SVG 叠加层，但不会自动应用。用户可以选择“使用检测结果”创建 `automatic` selection，也可以选择“调整检测区域”进入手动校正。低置信度和无效候选会显示失败说明，并只允许用户手动拖出近似外框；低置信度永远不会被接受为自动选择。

## 整图像素翻转与网格单元位置镜像的区别

普通的整图像素翻转会把图片每一个像素按左右方向反转。这样虽然视觉上完成了镜像，但图片中的文字、标签和内部标记也会一起被反向翻转，例如 `H7` 会变成镜像文字。

Mirror Master 计划实现的是网格单元位置镜像：先理解图片由哪些网格单元组成，再把这些单元在网格坐标中的位置做镜像排列。每个单元内部的图像内容和标签保持原始方向，因此标签仍应可读。

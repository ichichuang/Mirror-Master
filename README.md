# 豆图设计台

豆图设计台（仓库名保持 Mirror Master）是面向拼豆顾客的完整拼豆图纸设计工具，以结构化拼豆颜色矩阵为业务权威，支持照片、像素画和已有图纸三种模式。现有 OpenCV/Pillow 网格感知镜像能力保留为“已有图纸智能镜像”模块。

> [中文产品规范](docs/PRODUCT_SPEC.zh-CN.md) 是产品目标、算法、数据模型、体验和验收标准的唯一权威。其他文档仅提供实现或运维说明，不得定义竞争性产品规范。
>
> Vercel 在重建期间处于暂停状态；不得部署。生产运维背景参见 [中文部署指南](docs/DEPLOYMENT.zh-CN.md)。

## 目标流程

1. 选择照片、像素画或已有图纸模式并上传 PNG、JPEG 或 WebP。
2. 裁剪/旋转，设置行列、拼豆与拼板参数，选择真实可用色板。
3. 生成确定性的拼豆颜色矩阵，并核对材料、实际尺寸和拼板布局。
4. 在 Canvas 中编辑、撤销/重做、切换正反面或执行矩阵镜像。
5. 导出 PNG、PDF、CSV 或项目 JSON。

已有图纸模式继续使用网格检测与完整 cell 重排，保持网格外标题、坐标、标签和图例不变。

## 重建目标架构

- Vite/TypeScript：移动端优先的上传、裁剪、生成工作区、Canvas 编辑器、palette/material inspector 和结果导出。
- FastAPI/OpenCV/Pillow：图片解码、确定性生成、严格项目校验、导出，以及保留的已有图纸网格检测/智能镜像。
- 结构化 palette source：生成前后端机器资产并验证默认 39 色与 MARD 221 色。
- 项目 `cells` 矩阵：预览、统计、材料、镜像和导出的唯一业务真相。

## 本地启动

要求 Python 3.12，以及 pnpm 或 Corepack。脚本会创建或复用 `backend/.venv`、安装依赖、构建前端，并在不启用 reload 的情况下启动统一服务：

```bash
./scripts/start-local.sh
```

应用地址：`http://127.0.0.1:8000`

健康检查：`http://127.0.0.1:8000/api/health`

仅开发前端时，Vite 会把 `/api` 代理到 `http://127.0.0.1:8000`。

## Docker 部署

```bash
docker compose up -d --build
```

容器通过 `0.0.0.0:8000` 提供统一前后端服务，宿主机映射端口为 `8000`。

## 隐私

图片会上传到用户控制的豆图设计台服务，在内存中完成解码、识别、镜像和 PNG 编码，不写入持久存储，也不发送给第三方。服务不记录文件名、图片字节、哈希或边界合同，并为响应设置 `Cache-Control: no-store`。部署方仍应使用 HTTPS，并确认代理和平台日志不会记录请求体。
